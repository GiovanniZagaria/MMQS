Creating a new Analysis

  • First, create a new analysis file with the CARiSMA wizard (File → New → Other and den choose CARiSMA → Analysis, or File → New → Analysis if you are in the CARiSMA Perspective). Enter the filename (file extension must be .adf) and a folder to create the new file and click Next.


  • Choose the model type. Then select the model file which should be analyzed and click Next.



    • Figure: Analysis Wizard - Page 1

  • The analysis-file-name is predefined and will be "modelname"_analysis, but you can change this name if you want to.
    By default, the analysis file will be stored in the folder where the model lies that will be analyzed.
    Now click     Finish to create the analysis file.
    The new analysis is automatically opened and shown in the editor view.



    • Figure: Analysis Wizard - Page 2

Figure: The elements of the CARiSMA GUI

The Analysis Editor

The Analysis Editor helps you to edit the settings of an analysis for your model.

Performing an Analysis

  • Add one or more checks to the analysis by clicking on the Add button and selecting one or more from the listed checks.
  • Configure the parameters of a check by highlighting it in the Selected checks list. The parameters for the check are shown on the right side of the editor window.
  • Required parameters can be identified by an asterisk.
  • After entering appropriate values for all parameters, click Run... to start the analysis.
  • The results will be shown in the Analysis Results view.
As shown, the upper left part of the Analysis Editor consists of information about the analysis: its name, the used model file, the associated editor and its model type. The name and the model file are editable. The Browse button allows you to change the used model file. The Open Model button opens the model file in an editor. To edit the settings of the Open Model button, go to the preferences of CARiSMA.

The Analysis Results View

After executing an analysis, its results or rather the results of the checks used in the analysis is shown in the Analysis Results view.

    Figure: The Analysis Results view
  • The view is organized as a treeview.
  • Each root element represents the run of one analysis.
  • Second level entries correspond to the checks performed during the analysis.
  • The last level shows the result messages of a check. The icon in the beginning can be one of these types: Error, Warning or Information.

Creating a report

Each check can create an extensive text report. Right-click on any analysis entry in the Analysis Results view and select Create report for selected analysis from the context menu to generate the report for the analysis. The report will be automatically saved inside the folder containing the .adf file.
